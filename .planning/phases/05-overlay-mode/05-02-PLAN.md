    **A. Create OverlayPanelView:**
    A class that wraps ComposeView in a WindowManager overlay window with drag, resize, pin/lock toggles. Constructor takes:
    ```kotlin
    class OverlayPanelView(
        private val displayContext: Context,
        private val windowManager: WindowManager,
        private val translations: StateFlow<List<TranslationEntry>>,
        private val onPlayAudio: (String) -> Unit,
        private val onWordLookup: (suspend (SegmentedWord) -> List<DictionaryResult>)?,
        private val initialConfig: OverlayConfig,
        private val onConfigChanged: (OverlayConfig) -> Unit
    )
    ```

    Implementation:
    1. `fun show()`: Create ComposeView using displayContext. Set content to DsTranslatorPresentationTheme wrapping a panel composable that contains:
       - A top bar with: minimize button (X icon, calls onMinimize callback), pin toggle (icon changes based on isPinned state), lock toggle (icon changes based on isLocked -- use a padlock icon). Top bar has a semi-transparent dark background.
       - The body: `TranslationListScreen` composable reused directly from existing code, passing translations, onPlayAudio, onWordLookup.
       - A resize handle in the bottom-right corner (small triangle/grip indicator).
       Create PresentationLifecycleOwner, dispatch ON_CREATE/ON_START/ON_RESUME. Set tree owners.
       Create WindowManager.LayoutParams:
       - width = initialConfig.panelWidth, height = initialConfig.panelHeight
       - TYPE_APPLICATION_OVERLAY
       - FLAG_NOT_FOCUSABLE | FLAG_SECURE (NOT FLAG_NOT_TOUCHABLE -- panel needs touch input when unlocked)
       - gravity = TOP | START, x = initialConfig.panelX, y = initialConfig.panelY
       - alpha = initialConfig.panelAlpha
       Call windowManager.addView(composeView, params).

    2. Drag handling: Attach touch listener to the top bar area. On ACTION_DOWN record initial position + touch coordinates. On ACTION_MOVE update params.x/y and call windowManager.updateViewLayout(). On ACTION_UP call onConfigChanged with updated position. Use same DRAG_THRESHOLD=10 pattern from FloatingButtonService.

    3. Resize handling: Attach touch listener to the resize handle. On ACTION_MOVE, calculate new width/height from drag delta (minimum 200x150 dp). Update params.width/height and call windowManager.updateViewLayout(). On ACTION_UP call onConfigChanged with updated size.

    4. `fun setLocked(locked: Boolean)`: When locked, add FLAG_NOT_TOUCHABLE to params and call updateViewLayout -- panel becomes visible but non-interactive. When unlocked, remove FLAG_NOT_TOUCHABLE and updateViewLayout. Update internal locked state for UI indicator.

    5. `fun hide()`: Dispatch ON_PAUSE/ON_STOP/ON_DESTROY on lifecycleOwner BEFORE calling safeRemoveView. Null out composeView reference.

    6. `var onMinimize: (() -> Unit)? = null` -- callback for minimize button tap.

    7. Alpha must be <= 0.8f for Android 12+ untrusted touch compliance when touch passthrough is needed. When locked (passthrough), clamp alpha to min(config.panelAlpha, 0.8f).

    **B. Create OverlayTooltip:**

    ```kotlin
    @Composable
    fun OverlayTooltip(
        word: SegmentedWord,
        dictionaryResult: DictionaryResult?,
        jlptLevel: Int?,
        onPlayAudio: (String) -> Unit,
        onDismiss: () -> Unit,
        modifier: Modifier = Modifier
    )
    ```
    Content: Small rounded card (dark background, white text) showing:
    - Word surface form + reading (if available from dictionaryResult)
    - JLPT level badge (colored, e.g., N5=green, N1=red) -- reuse existing JLPT color scheme
    - First definition (truncated to 2 lines)
    - Small audio button (speaker icon)
    - Tap outside dismisses

    Position logic (Claude's discretion): The tooltip should be positioned near the tapped element. Use Popup composable with IntOffset. If the calculated position would go off-screen, flip to above/below or left/right. Keep within safe margins (16dp from edges).

    **A. Create OverlaySourceLabels:**
    Manages Canvas-drawn translation labels positioned at OCR bounding box screen coordinates. Uses lightweight View subclass (not ComposeView) since there may be many small labels.

    ```kotlin
    class OverlaySourceLabels(
        private val displayContext: Context,
        private val windowManager: WindowManager,
        private val onLabelTap: (OcrTextBlock, TranslationEntry) -> Unit
    )
    ```

    Implementation:
    1. Inner class `SourceLabelView(context: Context) : View(context)` that:
       - Draws semi-transparent dark background (Color.argb(180, 30, 30, 30))
       - Draws white text (translation English text) with anti-aliased Paint, textSize configurable
       - Has auto-offset option: if enabled, shifts label 2-4dp below the bounding box so original Japanese is visible above
       - Overrides onTouchEvent: on ACTION_UP, calls onLabelTap callback
       - Uses onDraw(Canvas) with two Paint objects (bgPaint and textPaint), pre-measured text width/height for sizing

    2. `fun showLabels(ocrResult: OcrResult, translations: List<TranslationEntry>)`:
       - Clear any existing labels first (call clearLabels())
       - For each OcrTextBlock in ocrResult.textBlocks:
         - Map to screen coordinates using OverlayCoordinateMapper.mapToScreenCoordinates(block, ocrResult)
         - Find the matching TranslationEntry by matching block.text to entry.japanese
         - If no match, skip this block
         - Create SourceLabelView, set translation text
         - Create WindowManager.LayoutParams: TYPE_APPLICATION_OVERLAY, FLAG_NOT_FOCUSABLE | FLAG_SECURE, width = mapped rect width (min 100px), height = WRAP_CONTENT, gravity = TOP|START, x = mapped rect.left, y = mapped rect.bottom (auto-offset: add 4dp gap below original text)
         - Alpha = 0.8f (Android 12+ compliant for passthrough)
         - Call windowManager.addView(labelView, labelParams)
         - Store in labels: MutableList<Pair<View, WindowManager.LayoutParams>>
       - Apply fade-in animation (ObjectAnimator on alpha from 0 to target over 200ms) per Claude's discretion

    3. `fun clearLabels()`: Iterate labels list, safeRemoveView each. Clear the list. Apply brief fade-out before removal (150ms) if labels currently visible.

    4. `fun setTouchable(touchable: Boolean)`: For each label, toggle FLAG_NOT_TOUCHABLE on/off and call updateViewLayout. When not touchable (locked mode), all label touches pass through to game.

    **B. Create OverlayDisplayManager:**
    Central manager owning all overlay windows for a target display.

    ```kotlin
    class OverlayDisplayManager(
        private val service: Service,
        private val targetDisplay: Display,
        private val translations: StateFlow<List<TranslationEntry>>,
        private val ocrResults: StateFlow<OcrResult?>,
        private val onPlayAudio: (String) -> Unit,
        private val onWordLookup: (suspend (SegmentedWord) -> List<DictionaryResult>)?,
        private val settingsRepository: SettingsRepository
    )
    ```

    Implementation:
    1. Constructor: Get WindowManager via `service.createDisplayContext(targetDisplay).getSystemService(WINDOW_SERVICE) as WindowManager`. Store displayContext.

    2. Properties:
       - `private var panelView: OverlayPanelView? = null`
       - `private var sourceLabels: OverlaySourceLabels? = null`
       - `private var currentMode: OverlayMode = OverlayMode.Off`
       - `private var isPinned = false`
       - `private var isLocked = false`
       - `private val scope = CoroutineScope(SupervisorJob() + Dispatchers.Main)`

    3. `fun showPanel(config: OverlayConfig)`:
       - Create OverlayPanelView with displayContext, windowManager, translations, callbacks, config
       - Set onMinimize callback to call hidePanel() (or just collapse if pinned)
       - Set onConfigChanged to persist via settingsRepository.setOverlayConfig()
       - Call panelView.show()
       - currentMode = OverlayMode.Panel

    4. `fun hidePanel()`:
       - panelView?.hide()
       - panelView = null

    5. `fun showSourceLabels()`:
       - Create OverlaySourceLabels with displayContext, windowManager, onLabelTap callback
       - Start collecting ocrResults flow in scope:
         ```kotlin
         scope.launch {
             ocrResults.collect { result ->
                 if (result != null) {
                     sourceLabels?.showLabels(result, translations.value)
                 }
             }
         }
         ```
       - currentMode = OverlayMode.OverlayOnSource

    6. `fun hideSourceLabels()`:
       - sourceLabels?.clearLabels()
       - sourceLabels = null

    7. `fun switchMode(mode: OverlayMode)`:
       - Use OverlayStateMachine.transition(currentMode, mode) to determine cleanup
       - If cleanup needed for OverlayOnSource: call hideSourceLabels()
       - If cleanup needed for Panel: call hidePanel()
       - Then show the new mode:
         - OverlayOnSource: showSourceLabels()
         - Panel: showPanel(load config from settingsRepository using isPrimary based on display ID)
         - Off: nothing (both already cleaned)
       - currentMode = mode

    8. `fun setLocked(locked: Boolean)`:
       - isLocked = locked
       - panelView?.setLocked(locked)
       - sourceLabels?.setTouchable(!locked)

    9. `fun setPinned(pinned: Boolean)`:
       - isPinned = pinned

    10. `fun cleanup()`:
        - hidePanel()
        - hideSourceLabels()
        - scope.cancel()
        - currentMode = OverlayMode.Off
        - Log.d("OverlayDisplayManager", "All overlay views cleaned up for display ${targetDisplay.displayId}")

    11. The onLabelTap callback for source labels should: create a small overlay tooltip window near the tapped label position showing word details (this will use a ComposeView with OverlayTooltip composable). Dismiss the tooltip on outside tap or after 5 seconds timeout.

    Ensure ALL addView calls use try-catch for IllegalStateException (view already added). Ensure ALL removeView calls use try-catch for IllegalArgumentException (view already removed). This is critical for the "no leaked views" requirement.
  </action>
