    **B. Expand FloatingButtonService with overlay mode integration:**

    Add new member variables:
    ```kotlin
    private lateinit var btnOverlayMode: ImageView
    private lateinit var btnScreenSwitch: ImageView
    private var overlayDisplayManager: OverlayDisplayManager? = null
    private var currentOverlayMode: OverlayMode = OverlayMode.Off
    private var currentDisplayId: Int = Display.DEFAULT_DISPLAY
    ```

    In onCreate():
    1. Find new view references: btnOverlayMode, btnScreenSwitch
    2. Add overlay mode button click listener:
       ```kotlin
       btnOverlayMode.setOnClickListener {
           observerScope.launch {
               // Cycle: Off -> OverlayOnSource -> Panel -> Off
               val nextMode = when (currentOverlayMode) {
                   OverlayMode.Off -> OverlayMode.OverlayOnSource
                   OverlayMode.OverlayOnSource -> OverlayMode.Panel
                   OverlayMode.Panel -> OverlayMode.Off
               }
               handleOverlayModeSwitch(nextMode)
           }
           collapseMenu()
       }
       ```
    3. Add screen switch button click listener:
       ```kotlin
       btnScreenSwitch.setOnClickListener {
           observerScope.launch { handleScreenSwitch() }
           collapseMenu()
       }
       ```
    4. Add overlay mode state observer to update button background:
       ```kotlin
       observerScope.launch {
           settingsRepository.overlayModeFlow().collect { mode ->
               currentOverlayMode = mode
               val bgRes = if (mode != OverlayMode.Off) R.drawable.bg_floating_button_active else R.drawable.bg_floating_button
               btnOverlayMode.setBackgroundResource(bgRes)
           }
       }
       ```

    Add `handleOverlayModeSwitch(mode: OverlayMode)`:
    1. If mode is Off: cleanup existing overlay, set SettingsRepository overlay mode to Off. If Presentation was dismissed, send ACTION_RESTORE_PRESENTATION to CaptureService.
    2. If mode is OverlayOnSource or Panel:
       - If overlayDisplayManager is null, create one using the current display (get from DisplayManager). Pass CaptureService.translations, CaptureService.latestOcrResult, TTS callback (send ACTION to CaptureService via intent), dictionary lookup (null for now -- overlay shows translation-only).
       - Call overlayDisplayManager.switchMode(mode)
       - Persist via settingsRepository.setOverlayMode(mode)
       - Optionally dismiss Presentation: if settingsRepository.getDismissPresentationOnOverlay() is true, send ACTION_DISMISS_PRESENTATION intent to CaptureService

    Add `handleScreenSwitch()`:
    1. Get DisplayManager, find all displays
    2. Determine target display: if currently on default display, switch to secondary. If on secondary, switch to default.
    3. Clean up current overlay: overlayDisplayManager?.cleanup(), overlayDisplayManager = null
    4. Remove bubble from current WindowManager: windowManager.removeView(menuView)
    5. Create new display context and WindowManager for target display:
       ```kotlin
       val targetDisplay = displayManager.displays.first { it.displayId != currentDisplayId }
       val targetContext = createDisplayContext(targetDisplay)
       windowManager = targetContext.getSystemService(WINDOW_SERVICE) as WindowManager
       ```
    6. Re-add menuView to new WindowManager with restored position (load per-screen position from settingsRepository, or use default)
    7. Update currentDisplayId
    8. If overlay mode is not Off, recreate OverlayDisplayManager on the new display and re-show overlay

    Update expandMenu() and collapseMenu():
    - Add btnOverlayMode and btnScreenSwitch to the buttons list
    - expandMenu: all 8 buttons shown with staggered animation (two rows)
    - collapseMenu: all 8 buttons hidden with reverse animation

    Update onDestroy():
    - Call overlayDisplayManager?.cleanup() before removing menuView
    - Set overlayDisplayManager = null

    **C. Add Presentation dismiss support to CaptureService:**
    1. Add new constant: `const val ACTION_DISMISS_PRESENTATION = "com.dstranslator.action.DISMISS_PRESENTATION"`
    2. In onStartCommand(), add case for ACTION_DISMISS_PRESENTATION: `presentation?.dismiss(); presentation = null`
    3. Add new constant: `const val ACTION_RESTORE_PRESENTATION = "com.dstranslator.action.RESTORE_PRESENTATION"` -- when overlay mode goes Off, restore Presentation if dismissed
    4. In onStartCommand(), add case for ACTION_RESTORE_PRESENTATION: if presentation is null, call setupPresentation()

    **State cleanup verification** -- ensure all transition paths are covered:
    - OverlayOnSource to Panel: overlayDisplayManager.switchMode handles cleanup (hides source labels before showing panel)
    - Panel to OverlayOnSource: overlayDisplayManager.switchMode handles cleanup (hides panel before showing labels)
    - Any mode to Off: overlayDisplayManager.cleanup() removes everything
    - Screen switch: cleanup() on old display, recreate on new display
    - Service onDestroy: cleanup() called
    - Profile switch: FloatingButtonService doesn't own profiles, but overlay config refresh should happen via settingsRepository Flow observation
